# PropEditorX

## Vision
A modern, i18n-focused `.properties` file management plugin for the Eclipse IDE that provides a "Lokalise-style" user experience. It aims to serve all locales in a single, multi-mode editor, enhancing productivity for developers, translators, and stakeholders through superior UX and DX.

## Core Goals
- **Multi-Mode Editor**: Complete the "Hero" feature that provides a unified, side-by-side editing experience for all locale properties.
- **Developer Experience (DX)**: Ensure a smooth, responsive, and intuitive workflow within the Eclipse environment.
- **User Experience (UX)**: Deliver a top-tier translation management interface comparable to industry leaders.

## Target Audience
Java developers working on the Eclipse platform who need to manage complex internationalization (i18n) projects.

## Technical Constraints & Standards
- **Java Version**: Built with modern Java (currently Java 25), maintaining backward compatibility down to Java 17 (LTS).
- **Performance**: High priority on responsiveness and minimal overhead on the Eclipse IDE.
- **Toolkit**: Built with native SWT and official Eclipse platform APIs.
- **Architecture**: Emphasis on clean code, SOLID principles, and mainstream Java implementation patterns accepted by the community.
- **Internationalization**: Support for human-readable UTF-8 text without Unicode escaping.

## Stakeholders
- **Xenogew**: Lead developer and project owner.
- **Java Developers/Translators**: End-users of the plugin.
